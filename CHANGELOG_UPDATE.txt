The Hermit v1.1.0 — Major Card Accuracy & Bug Fix Update

This is a large update that brings every card in line with The Hermit v1.56 — the final version from DownfallSTS (STS1). A full audit of all 75+ cards was performed by comparing our STS2 implementations against the original Java source code on the DownfallSTS GitHub repository. In addition to card accuracy, several gameplay-breaking bugs have been fixed.

Many thanks to the community members who took the time to report issues. You are all credited at the bottom of this post.

=============================================
BUG FIXES
=============================================

1. SPRAY N' PRAY — NOT SHUFFLING DOUBT, WRONG TARGETING (reported by berserker2111)

   Previously, Spray n' Pray dealt damage to ALL enemies and did not shuffle a Doubt into the draw pile. Both of these were wrong.

   Fixed: The card now deals damage to random enemies (3 hits) and correctly shuffles a Doubt into the draw pile after dealing damage. Rarity was also corrected from Uncommon to Common.

2. CHEAT — DOES NOTHING WHEN PLAYED (reported by Goodcat33)

   The Cheat card let you pick a card from the top of your draw pile, but then nothing happened. The card you picked just vanished.

   Root cause: The selected card was still in the draw pile when AutoPlay was called. STS2's AutoPlay only works on cards that are in the player's hand.

   Fixed: The selected card is now moved to the player's hand first, then auto-played. Works in both single player and multiplayer.

3. ETERNAL FORM — AFFECTS HOST INSTEAD OF HERMIT IN MULTIPLAYER (reported by Goodcat33)

   In multiplayer, if the Hermit wasn't the host, Eternal Form's cost reduction was being applied to the host character's drawn cards instead of the Hermit player's cards.

   Root cause: The AfterCardDrawn hook fires for ALL players' cards, not just the power owner's. There was no check to verify the drawn card belonged to the Hermit player.

   Fixed: Added an owner check — `if (card.Owner != Owner.Player) return;` — so the cost reduction only applies to the Hermit's own cards.

4. DENTED PLATE RELIC — NO HP CHECK, MISSING CARD DRAW (reported by Condomknight)

   The Dented Plate relic was granting +1 Energy at the start of every turn regardless of HP. It's supposed to only activate when you're at or below 50% HP, and it should also draw a card.

   Fixed: Added the HP <= 50% check (`creature.CurrentHp > creature.MaxHp / 2`). Also added the missing card draw. Also switched from AfterEnergyReset to BeforeSideTurnStart hook so it has proper context for drawing cards.

5. EPOCH/TIMELINE STUCK BUG — PHANTOM EPOCHS AFTER FIRST WIN (reported by rekij)

   After beating the first run with Hermit, 4 epochs appeared on the timeline to reveal. After going through them, one would not register as obtained, leaving you stuck in the timeline screen unable to start a new run.

   Root cause: The epoch slot initializer was calling UnlockSlot() for ALL 6 Hermit epochs on every game load, which created phantom "NotObtained" entries for epochs the player hadn't actually earned yet. These showed up as locked epochs with no way to unlock them.

   Fixed: The initializer now only promotes epochs that are in the "ObtainedNoSlot" state (meaning the player actually earned them but they didn't have a timeline slot yet). Epochs that haven't been obtained are left alone.

6. CONCENTRATION POWER — STACK DECREMENT ERROR

   When a Concentration stack was consumed (e.g., by playing a Dead On card with Concentration active), the game would hit a read-only property error because PowerModel.Amount can't be set directly from outside the class.

   Fixed: Added a public ConsumeStack() method on ConcentrationPower that calls the protected SetAmount() internally, then removes the power if stacks hit 0.

=============================================
KNOWN LIMITATIONS & STS2 ADAPTATIONS
=============================================

Some STS1 mechanics don't have direct equivalents in STS2. These cards have been adapted as faithfully as possible:

1. PLATED ARMOR DOES NOT EXIST IN STS2
   Plated Armor was an STS1 buff that granted Block at the end of every turn and lost 1 stack when you took unblocked damage. STS2 does not have this mechanic.

   Affected cards:
   - Dive: Dead On originally granted 1(2) Plated Armor. Now grants 4(6) bonus Block instead (one-time, not recurring). The Block values are higher to compensate for losing the recurring nature.
   - Scavenge: Originally granted 4(5) Plated Armor. Now grants 12(15) Block instead. Values scaled up significantly since Block is one-time vs. Plated Armor's recurring benefit.

   If there's community interest, we may implement a custom Plated Armor power in a future update.

2. ENHANCED STRIKE GENERATION (High-Caliber)
   In STS1, High-Caliber generated a Strike with +2(+3) bonus damage. STS2 doesn't easily support creating modified copies of existing cards with stat bonuses, so a normal Strike is generated instead. The base damage of High-Caliber itself (6/9) partially compensates.

3. GOLD GAIN (Scavenge Dead On)
   Scavenge's Dead On effect grants gold using PlayerCmd.GainGold. This API call is unverified — if the gold gain doesn't work in-game, please report it and we'll find an alternative.

4. DESPERADO SELF-SCALING
   Desperado's "double damage and increase cost" mechanic uses STS2's DynamicVar and EnergyCost APIs. The scaling should persist for the rest of combat, but if the numbers behave unexpectedly (e.g., not persisting between turns, or stacking incorrectly), please let us know.

5. FROM BEYOND — HP LOSS VS. DAMAGE
   From Beyond uses unblockable/unpowered damage to simulate STS1's HP loss (FastLoseHPAction). This means it bypasses Block, which is faithful to the original. However, it may interact differently with damage-modifying effects than in STS1.

6. HEROIC BRAVADO — COST REDUCTION
   Heroic Bravado reduces its own cost each time it's played. The STS2 API for permanent cost modification (SetCustomBaseCost) may behave slightly differently from STS1's ReduceCostActionFixed. If the cost reduction doesn't persist correctly between turns, please report it.
